About the Dataset

The dataset used for this project is a collection of over 5,500 messages, classified as either legitimate ("ham") or "spam." This dataset is used to train and evaluate the performance of our email classifier.

Data Preprocessing

  • Text normalization and cleaning
  • Removing duplicate messages
  • Removing common English stop words
  • Tokenization and feature extraction using CountVectorizer
  • Label encoding for classification

Data Split

  • Training: 80% (4458 messages)
  • Testing: 20% (1114 messages)
  • Stratified sampling maintained
  • Cross-validation performed

Feature Engineering

Our models extract various features from email text to identify spam characteristics:

Text Features

  • Word frequency (TF-IDF)
  • N-gram analysis
  • Character count
  • Word count
  • Sentence length

Content Features

  • URL presence and count
  • Email address patterns
  • Phone number detection
  • Currency mentions
  • Attachment indicators

Style Features

  • Capital letters ratio
  • Exclamation mark count
  • Special characters
  • HTML tag presence
  • Font formatting

Semantic Features

  • Spam keywords
  • Urgency indicators
  • Financial terms
  • Medical references
  • Promotional language
